## 数据集不是越多越好

在建筑外立面缺陷检测项目中，数据量并不是越多越好。如果不同数据集的拍摄距离、缺陷类别、标注粒度和图像风格差异很大，直接混在一起训练，模型可能学到的是数据集风格，而不是缺陷本身。

例如，同样叫“裂缝”，不同数据源可能差别很大：

- 有的数据是近距离裂缝局部图。
- 有的数据是墙面远景图。
- 有的数据只标裂缝主体。
- 有的数据把阴影和污渍也混进了缺陷类别。
- 有的数据框很紧，有的数据框偏大。

如果不做清洗，后续模型训练出现问题时，很难判断到底是模型不行、参数不行，还是数据本身有问题。

## 目标场景：外立面巡检

这个实习项目关注的是建筑外立面巡检，不是单纯的裂缝局部检测。数据整理时需要始终围绕最终场景：

- 图像来自无人机或类似视角。
- 缺陷在整张图中占比小。
- 背景包含窗户、墙砖、阴影、管线、空调外机等干扰。
- 检测结果后续要进入三维定位和报告系统。

因此数据整理更关注“数据能不能代表现场场景”，而不是单纯追求数量。

## 数据整理流程

数据集整理可以拆成五步：

```text
收集原始数据
  -> 可视化审查
  -> 剔除不可用样本
  -> 统一类别和格式
  -> 补充现场标注
  -> 拆分 train / val / test
```

### 1. 可视化审查

第一步不是训练，而是看标注。

很多数据问题只看文件名和数量看不出来，必须把框画到图上：

- 框是否明显偏移。
- 是否有大量漏标。
- 类别是否混乱。
- 框是否过大或过小。
- 是否存在空标签或损坏图片。

通常先写一个简单脚本批量生成预览图，再人工快速扫一遍，把明显不可用的样本剔除。

### 2. 剔除不可用样本

不可用样本包括：

- 图片模糊严重。
- 标注缺失或错位。
- 类别语义不清。
- 缺陷目标太小且无法确认。
- 和建筑外立面场景差距太大。

剔除样本不是浪费数据，而是减少噪声。对于小目标检测来说，错误标签对训练的影响很大，尤其是漏标会让模型把真实缺陷当成背景学习。

### 3. 统一类别

不同数据源的类别命名往往不统一。比如同一类问题可能叫 crack、裂缝、wall_crack，也可能把剥落、破损、脱落混在一起。

先确定项目需要的类别集合，再做映射：

```text
crack / 裂缝 / wall_crack -> 裂缝
spalling / 剥落 / 脱落 -> 剥落
rebar_exposure / 露筋 -> 露筋
efflorescence / 泛碱 -> 泛碱
```

类别不宜一开始拆得太细。类别越细，标注一致性要求越高。如果样本量不足，过细类别反而会降低模型稳定性。

### 4. 标注格式转换

项目中会遇到 XML、JSON、LabelMe JSON、YOLO txt 等多种标注格式。训练前需要统一到 YOLO 格式。

转换时要注意：

- 坐标是否为像素坐标。
- 框格式是 xyxy 还是 xywh。
- 是否需要归一化。
- 类别 id 是否从 0 开始。
- 图片尺寸是否正确读取。

### 标注质检和格式转换伪代码

```text
function buildTrainingLabels(dataset):
  cleanSamples = []

  for sample in dataset:
    image = loadImage(sample.imagePath)
    labels = parseAnnotation(sample.annotationPath)
    labels = normalizeCategoryNames(labels)

    preview = drawLabels(image, labels)
    reviewResult = humanReview(preview)

    if reviewResult == "reject":
      recordRejectReason(sample)
      continue

    yoloLabels = convertToYolo(labels, image.width, image.height)
    cleanSamples.append(image, yoloLabels)

  splitByScene(cleanSamples)
  exportYoloDataset(cleanSamples)
```

这段伪代码强调：数据进入训练之前，必须先经过可视化审查和类别统一。

## 为什么要补充现场标注

开源数据集能提供基础样本，但现场外立面图像仍然需要补标。原因是：

- 现场拍摄角度和开源数据不同。
- 缺陷尺度更接近最终推理场景。
- 背景纹理更复杂。
- 实际类别分布可能不均衡。

项目中使用 LabelMe 对部分现场图像进行补标，让训练和验证更接近项目真实数据。

## train / val / test 怎么拆

建筑外立面图片有一个容易忽略的问题：同一栋楼、同一次航拍、相邻照片之间非常相似。

如果随机拆分训练集和验证集，可能会出现这种情况：

- 训练集里有某一面墙的照片。
- 验证集里也有同一面墙的相邻照片。
- 模型验证指标看起来不错。
- 但换到另一栋楼或另一个拍摄角度后效果下降。

所以更合理的方式是按场景拆分：

- 按建筑拆分。
- 按拍摄批次拆分。
- 按楼面或立面方向拆分。

这样验证集更接近真实泛化能力。

## 数据整理要点

这部分工作看起来不像模型训练那么“显眼”，但它直接决定模型上限。关键要点包括：

- 先看数据，再谈模型。
- 标签质量比样本数量更重要。
- 类别定义要符合最终业务。
- 训练集和验证集不能只随机拆。
- 现场样本必须纳入闭环。
- 人工复核结果可以反哺数据集。

## 小结

建筑缺陷检测的数据准备不是简单把图片放进 dataset 文件夹。它需要围绕外立面巡检场景做一轮完整的数据治理：

```text
数据筛选
  -> 标注质检
  -> 类别统一
  -> 现场补标
  -> 场景化拆分
  -> 训练验证
```

如果这一步做好，后面的 SAHI、YOLO 训练、人工复核和三维定位才有稳定基础。
