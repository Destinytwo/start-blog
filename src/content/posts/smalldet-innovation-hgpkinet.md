## 问题背景

无人机小目标通常只占整张图像很小一部分，目标本身的边缘、纹理和形状线索很弱。模型在连续下采样后，原本只有几个像素宽的目标可能被压缩到特征图上的一个点，导致检测头很难判断这是目标、背景纹理还是噪声。

传统骨干网络更擅长提取大目标或中等目标的语义结构，但无人机航拍场景里常见的是车辆、行人、屋顶附属物、局部缺陷等小尺度对象。它们需要网络同时保留局部细节和一定范围的上下文：只看局部容易把背景纹理误判为目标，只看全局又容易忽略目标边界。

HG-PKINet 的设计出发点就是在特征提取阶段增强多尺度感知能力。它不是简单把网络堆得更深，而是在一个特征块内部组织不同感受野的分支，让局部纹理、中尺度结构和更大范围上下文能够共同参与表达。

## 模块结构怎么读

从模块图可以按照“输入特征、并行感受野、特征融合、残差输出”四步阅读。输入特征先被送入若干不同尺度的卷积分支，小尺度分支更关注边缘、角点和局部纹理，中尺度分支补充目标附近的结构信息，大尺度或大核分支扩大感受野，用来感知目标和周围背景之间的关系。

这些分支得到的特征不能直接简单相加，因为不同分支的响应强度和语义粒度不同。通常需要通过通道对齐、拼接、压缩或注意力式的重标定，让网络自动学习哪些尺度对当前样本更重要。最后再与原始输入形成残差路径，避免增强模块破坏已有表达。

这样做的价值在于，小目标不是只依赖一个尺度就能识别。比如远距离车辆可能只剩几个亮点，单靠局部纹理会像噪声；但结合道路区域、排列方式和周边上下文后，模型更容易判断它是否为有效目标。HG-PKINet 试图把这种多尺度判断放到骨干特征中提前完成。

- 增强不同尺寸小目标的特征响应
- 缓解单一卷积尺度对目标尺寸变化的不适应
- 为 Neck 特征融合提供更丰富的输入
- 通过残差结构降低模块插入后训练不稳定的风险
- 让骨干网络在浅层和中层阶段就保留更多可定位信息

## 伪代码拆解

下面的伪代码只表达模块思想，可以把它当作读图辅助：读者先理解数据流，再回到结构图里看每个分支的作用。

核心逻辑是先保留输入特征作为 shortcut，再让输入经过多个感受野分支，随后做融合和重标定，最后通过残差返回。这样写能说明为什么模块既要增强表达，又不能让网络失去原有稳定性。

### HG-PKINet 模块伪代码

```text
function HG_PKI_Block(feature):
  shortcut = feature

  localDetail = small_kernel_branch(feature)
  midContext = middle_kernel_branch(feature)
  wideContext = large_kernel_branch(feature)

  multiScale = concatenate(localDetail, midContext, wideContext)
  calibrated = channel_reweight(multiScale)
  enhanced = pointwise_fusion(calibrated)

  return normalize(shortcut + enhanced)
```

small_kernel_branch 表示局部细节分支，负责保留小目标边缘和纹理。

large_kernel_branch 表示更大感受野分支，负责补充目标周边上下文。

channel_reweight 和 pointwise_fusion 只表达融合思想，不展开具体参数、层数和配置。

## 训练和调试关注点

插入多尺度模块后，不能只看最终 mAP。更重要的是观察训练是否稳定、收敛是否变慢、显存是否明显上升，以及小目标类别的召回是否真的改善。如果总体指标提升但小目标召回没有提升，说明模块可能只是增强了中等目标或背景语义。

调试时可以把实验分成三个层次：第一，只替换骨干中的目标阶段，看参数量和速度变化；第二，和原始模型对比小目标类别的召回；第三，结合热力图或特征响应图观察模型是否更关注目标区域。这样比一次性堆多个模块更容易判断 HG-PKINet 的独立贡献。

在建筑缺陷检测场景中，这个思路也有迁移意义。裂缝、剥落、露筋等缺陷在高分辨率图像中经常表现为细线、小斑块或弱纹理，单尺度特征很难稳定识别。多尺度分支可以帮助模型同时捕获局部缺陷形态和墙面上下文。

- 看小目标类别 Recall，而不是只看总 mAP
- 记录参数量、推理耗时和显存变化，避免模块过重
- 用热力图检查增强后模型关注的位置是否更合理
- 做单模块消融，先证明骨干增强本身有效，再叠加 Neck 和 Loss
