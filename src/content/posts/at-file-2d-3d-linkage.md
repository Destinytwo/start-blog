## 为什么需要二维三维联动

这篇记录的是我在武汉市测绘研究院算法实习期间参与的外立面缺陷检测科研项目中的二维三维联动部分。

建筑外立面缺陷检测如果只停留在二维照片中，结果很难交付。用户能看到“某张照片上有裂缝”，但不容易知道这个缺陷在建筑物的哪个空间位置。

因此项目需要把二维照片中的检测框和三维模型中的建筑表面联系起来。这样系统才能支持：

- 在三维模型上显示缺陷红点。
- 点击三维点位时找到相关照片。
- 报告中按空间位置组织缺陷。
- 多张照片中的重复缺陷合并。

这一步的桥梁就是空三成果。空三文件提供相机参数、照片位姿和坐标关系，让二维像素和三维空间之间可以建立几何联系。

## 空三文件提供了什么

空三成果中通常包含以下信息：

- 相机内参：焦距、主点、像素尺寸等。
- 相机外参：相机位置和姿态。
- 照片位姿：每张照片在空间中的拍摄位置和方向。
- 坐标系统：工程坐标、局部坐标或投影坐标。
- 可能还包含畸变参数和照片索引信息。

对于联动来说，最核心的是两个问题：

1. 某个二维像素点对应相机视线中的哪条空间射线。
2. 这条射线与三维模型或建筑表面在哪里相交。

## 基本计算流程

简化流程如下：

```text
二维检测框
  -> 取框中心点或关键点
  -> 像素坐标归一化
  -> 转换到相机坐标系
  -> 根据相机姿态转到世界坐标系
  -> 生成空间射线
  -> 与模型表面求交
  -> 得到三维点坐标
```

### 二维像素到三维射线的伪代码

```text
function pixelToWorldRay(pixel, camera):
  normalized = inverse(camera.intrinsic) * [pixel.x, pixel.y, 1]
  cameraRay = normalize(normalized)

  worldRay = camera.rotation * cameraRay
  origin = camera.position

  return Ray(origin, worldRay)

function linkDefectToModel(defectBox, camera, model):
  center = centerPoint(defectBox)
  ray = pixelToWorldRay(center, camera)
  hit = intersect(ray, model.surface)

  return hit.worldCoordinate
```

这段伪代码只表达几何关系，真实项目中还要处理畸变、坐标原点、模型局部坐标和模型求交效率。

## 候选照片筛选

反过来，三维模型中的一个点也可以用于筛选候选照片。例如用户点击三维模型中的某个位置，系统可以根据相机位置和视角，找出可能拍到这个点的照片。

候选照片筛选可以减少无关图片参与计算，也方便人工复核。

常见判断条件包括：

- 三维点是否在相机视锥内。
- 点投影到照片后是否落在图像范围内。
- 相机朝向是否面向该点。
- 点和相机距离是否合理。
- 是否被模型表面遮挡。

## 坐标系统容易踩坑

二维三维联动最容易出问题的地方不是公式，而是坐标系统。

常见坑包括：

- 模型使用局部坐标，空三使用工程坐标。
- OSGB 存在 SRSOrigin，需要做原点偏移。
- 相机坐标轴方向和模型坐标轴方向不一致。
- 照片经过畸变校正，但检测框来自另一套图像。
- 切割模型后坐标范围发生变化。

所以我在系统中会把坐标转换拆开记录，避免写成一坨难以排查的公式。

```text
pixel coordinate
  -> normalized camera coordinate
  -> camera coordinate
  -> world coordinate
  -> model local coordinate
```

每一步都应能单独打印和验证。

## 工程验证方法

实现联动后不能只看代码是否能跑，还要做可视化验证：

1. 选取一张已知照片。
2. 在照片中选择一个明显建筑角点或缺陷点。
3. 计算空间射线和三维落点。
4. 在模型中显示该点。
5. 人工观察点位是否落在合理位置。

如果点位偏移，要逐步检查：

- 相机内参是否正确。
- 外参矩阵是否方向相反。
- 坐标原点是否漏加。
- 模型局部坐标是否未转换。
- 检测图和空三图是否同一尺寸。

## 小结

空三文件是二维照片和三维模型之间的桥梁。对于建筑外立面缺陷检测系统来说，它让检测结果从“照片里的框”变成“建筑表面的点”。

这一步的难点不只是数学公式，更在于：

- 坐标系统是否一致。
- 图像尺寸是否匹配。
- 模型原点是否正确。
- 切割模型是否破坏坐标链路。
- 结果是否经过可视化验证。

只有二维三维联动稳定，后续三维红点、缺陷去重、楼层切面和报告导出才有意义。
